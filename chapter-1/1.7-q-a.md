Q: Based on how you should name a variable, indicate whether each variable name is conventional (follows best practices), unconventional (compiler will accept but does not follow best practices), or invalid (will not compile), and why.</br>

a) ```int sum {};```</br>
(Assume it’s obvious what we’re summing)</br>
b) ```int _apples {};``` </br>
c) ```int VALUE {};```</br>
d) ```int my variable name {};```</br>
e) ```int TotalCustomers {};```</br>
f) ```int void {};```</br>
g) ```int numFruit {};```</br>
i) ```int 3some {};```</br>
j) ```int meters_of_pipe {};```</br>

A:</br>
a) conventional</br>
b) unconventional</br>
c) unconventional</br>
d) invalid, variables cannot have spaces</br>
e) conventional</br>
f) invalid, void is a keyword for standard c++</br>
g) conventional</br>
i) invalid, variable name cannot start with a number</br>
j) conventional</br>
