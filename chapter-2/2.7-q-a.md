#1</br>
Q: What is a function prototype?</br>
A: Function prototype is function's return type, function's identifier and function's arguments.</br>

#2</br>
Q: What is a forward declaration?</br>
A: Foward declaration tells the compiler that something exists without defining it.</br>

#3</br>
Q: How do we declare a forward declaration for functions?</br>
A: We decleare function's prototype above the function that needs to "see" it.</br>

#5</br>
Q: For each of the following programs, state whether they fail to compile, fail to link, or compile and link successfully. If you are not sure, try compiling them!</br>
a)</br>
```cpp
#include <iostream>
int add(int x, int y);

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int x, int y)
{
    return x + y;
}
```
b)</br>
```cpp
#include <iostream>
int add(int x, int y);

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int x, int y, int z)
{
    return x + y + z;
}
```
c)</br>
```cpp
#include <iostream>
int add(int x, int y);

int main()
{
    std::cout << "3 + 4 = " << add(3, 4) << '\n';
    return 0;
}

int add(int x, int y, int z)
{
    return x + y + z;
}
```
d)</br>
```cpp
#include <iostream>
int add(int x, int y, int z);

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int z, int y, int x) // names don't match the declaration
{
    return x + y + z;
}
```
e)</br>
```cpp
#include <iostream>
int add(int, int, int); // no parameter names

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}

int add(int x, int y, int z)
{
    return x + y + z;
}
```
f)</br>
```cpp
#include <iostream>

int add(int x, int y);

int add(int x, int y, int z)
{
    return x + y + z;
}

int main()
{
    std::cout << "3 + 4 + 5 = " << add(3, 4, 5) << '\n';
    return 0;
}
```

A:</br>
a) Will not compile, function takes 2 arguments, not 3.</br>
b) Will not compile, foward declaration has 2 arguments when function definition has 3.</br>
c) Will not link, there is no function add declaration that would take 2 arguments.</br>
d) Will compile and link</br>
e) Will compile and link</br>
f) Will compile and link</br>
