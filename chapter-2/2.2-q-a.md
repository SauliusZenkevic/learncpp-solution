Q: Inspect (do not compile) each of the following programs. Determine what the program will output, or whether the program will generate a compiler error.
(Assume you have “treat warnings as errors” turned off.)</br>

a)</br>
```cpp
#include <iostream>

int return7()
{
    return 7;
}

int return9()
{
    return 9;
}

int main()
{
    std::cout << return7() + return9() << '\n';

    return 0;
}
```
b)</br>
```cpp
#include <iostream>

int return7()
{
    return 7;

    int return9()
    {
        return 9;
    }
}

int main()
{
    std::cout << return7() + return9() << '\n';

    return 0;
}
```
c)</br>
```cpp
#include <iostream>

int return7()
{
    return 7;
}

int return9()
{
    return 9;
}

int main()
{
    return7();
    return9();

    return 0;
}
```
d)</br>
```cpp
#include <iostream>

int getNumbers()
{
    return 5;
    return 7;
}

int main()
{
    std::cout << getNumbers() << '\n';
    std::cout << getNumbers() << '\n';

    return 0;
}
```
e)</br>
```cpp
#include <iostream>

int return 5()
{
    return 5;
}

int main()
{
    std::cout << return 5() << '\n';

    return 0;
}
```
A:</br>
a) 16</br>
b) Error, nested functions are illegal.</br>
c) The program will not output anything.</br>
d) The program will output 5 on a new line each time, because return 7 never gets hit.</br>
e) The program will not compile, because it has an illegal name. Return is a c++ keyword.</br>

Q: What does “DRY” stand for, and why is it a useful practice to follow?</br>
A: DRY stands for "Don’t repeat yourself" and it states that solving a problem that requires doing something more than once, unnecessary redundance should be removed as much as possible.</br>
